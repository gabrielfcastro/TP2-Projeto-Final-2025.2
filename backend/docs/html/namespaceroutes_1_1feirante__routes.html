routes.feirante_routes.buscar_feirante (   id_feirante)

Busca um feirante pelo seu ID.

Esta rota processa uma requisição GET para recuperar os dados de um feirante específico com base no ID fornecido na URL.

Parameters
[in]id_feiranteID do feirante a ser buscado.
Postcondition
Busca o feirante na tabela 'feirantes' do banco de dados.
Exceptions
ExceptionPara erros internos do servidor durante a operação.
Returns
Retorna um JSON de resposta contendo:
  • Em caso de sucesso (200): Dicionário com os dados do feirante.
  • Em caso de erro (404): {"erro": "Feirante <id> não encontrado."}
  • Em caso de erro (500): {"erro": "Erro interno no servidor"}

routes.feirante_routes.criar_feirante ( )

Cria um novo feirante no sistema.

Esta rota processa uma requisição POST proveniente do front contendo os dados necessários para criar um novo feirante. O usuário deve estar autenticado para realizar esta operação.

Parameters
[in]nome_estabelecimentoNome do estabelecimento do feirante.
[in]link_wppLink do WhatsApp do feirante.
Precondition
nome_estabelecimento Deve ser uma string não vazia.
link_wpp Deve ser uma string não vazia.
Postcondition
Insere um novo feirante na tabela 'feirantes' do banco de dados.
Exceptions
ValueErrorSe os dados fornecidos forem inválidos.
LookupErrorSe o usuário autenticado não for encontrado no banco.
ConnectionErrorSe houver falha ao salvar o feirante no banco.
Returns
Retorna um JSON de resposta contendo os dados do feirante criado em caso de sucesso (201) ou uma mensagem de erro em caso de falha.

routes.feirante_routes.deletar_feirante (   id_feirante)

Deleta um feirante do sistema.

Esta rota processa uma requisição DELETE para remover um feirante específico do banco de dados com base no ID fornecido na URL. O usuário deve estar autenticado para realizar esta operação.

Parameters
[in]id_feiranteID do feirante a ser deletado.
Precondition
id_feirante Deve existir no banco de dados.
Postcondition
Remove o feirante da tabela 'feirantes' do banco de dados.
Exceptions
ValueErrorSe o feirante com o ID fornecido não for encontrado.
ExceptionPara erros internos do servidor durante a operação.
Returns
Retorna um JSON de resposta contendo:
  • Em caso de sucesso (200): {"mensagem": "Feirante <id> deletado com sucesso."}
  • Em caso de erro (404): {"erro": "Feirante <id> não encontrado."}
  • Em caso de erro (500): {"erro": "Erro interno no servidor"}
